Basics

1) Define each of the following.

  1. entry/datum: an item contained inside of a node
  2. node: a single "circle" or vertex in a tree that contains a item (the node is the container, the datum is the item inside that container).
  3. child: a node that has a parent (i.e. a node that branches off from another node)
  4. parent: a node that has at least one child (i.e. a node that has other nodes branching off from it)
  5. leaf: a node that has no children (has no other nodes branching off underneath it)
  6. forest: one or more trees
  7. binary tree: a type of tree that satisfies the following conditions
    • Each node can have at most 2 children, called a left and a right
    • Every element in the subtree to the left of the node must be smaller than the element in the node
    • Every element in the subtree to the right of the node must be larger than the element in the node

2) Which of the following are valid Tree constructors?

  1. Tree() Invalid: trees must contain at least one element, its datum.
  2. Tree(3) Valid: this constructs a leaf with 3 as its entry (notice that our implementation gives a default value for left and right)
  3. Tree(5, Tree(1), Tree(5)) Valid: this constructs a tree whose datum is 5, and has two children whose elements are 1 and 5.
  4. Tree(4, Tree(2)) Valid: This is a Tree that does not have a right child
  5. Tree(2, Tree(2, 4, 5)) Invalid: the 2nd and 3rd argument to a Tree constructor must be other Trees
