Basics

1) What type of object can first be? What type of object can rest be?

  1. Anything can be the first, even Rlists
  2. Only Rlists can be the rest of an Rlist

2) Which of the following are valid Rlist constructors?

  1. Rlist(1, 3) Invalid: rest must be an Rlist
  2. Rlist(1, None) Invalid: though this will technically work, it is violating abstraction because we can't assume empty is always defined as None
  3. Rlist(1, Rlist(4)) Valid
  4. Rlist(1) Valid
  5. Rlist() Invalid: Rlists must have a first

3) Constuct an rlist that contians the following elements:

    1, 3, lambda x: x, "hello"
Answer: 
    Rlist(1, Rlist(3, Rlist(lambda x: x, Rlist("hello"))))

4) What is the element at index 2 of the following Rlist?

    Rlist('this', Rlist('is', Rlist('an', Rlist('rlist', Rlist('question')))))
Answer: 
    'an'

5) What is the length of the Rlist in Q4?

    5

What would Python print?

    1) 1
    2) Rlist(2, Rlist(3))
    3) Rlist(3)
    4) 2
    5) 3
    6) Rlist(3, Rlist(2, Rlist(3)))
    7) Rlist(3, Rlist(4, Rlist(3)))
    8) Rlist(3, Rlist(4, 5))
    9) Invalid, rest must be an Rlist

Code writing

1) Implement tup_to_rlist.

    def tup_to_rlist(tup):
        tmp = rlist = Rlist(tup[0])
        for elem in tup[1:]:
            tmp.rest = Rlist(elem)
            tmp = tmp.rest
        return rlist

2) Implement rlist_to_list.

    def rlist_to_list(rlist):
        lst = []
        while rlist != empty:
            lst.append(rlist.first)
            rlist = rlist.rest
        return lst

3) Implement map_rlist.

    def map_rlist(rlist, f):
        while rlist != empty:
            rlist.first = f(rlist.first)
            rlist = rlist.rest
