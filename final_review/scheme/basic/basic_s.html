Code-writing

Q5

    (define (filter pred lst)
        (cond ((null? lst) nil)
              ((pred (car lst)) 
                  (cons (car lst) 
                        (filter pred (cdr lst))))
              (else (filter pred (cdr lst)))))

Explanation: If either lst is null, just return the empty list. If lst is not empty and its first item satisfies pred, place at the start of the new list, and filter the rest of the list. Otherwise, leave the first item out and just filter the rest of the list.

Q6

    (define (interleave list1 list2)
        (if (or (null? list1) (null? list2))
            (append list1 list2)
            (cons (car list1) 
                  (cons (car list2) 
                        (interleave (cdr list1) (cdr list2))))))

Explanation: If either list1 or list2 is empty, return the other one (appending an empty list to a non-empty list will just return the non-empty list). If neither list is empty, return the first item of list1, followed by the first item of list2, followed by the recursive call.

Q7

    (define (count-stairways n)
        (cond ((= n 1) 0)
            ((= n 2) 1)
            (else (+ (count-stairways (- n 1)) 
                     (count-stairways (- n 2))))))

Explanation: If there is one stair left, there is only 1 way to reach the top (take 1 step). If there are two stairs left, there are 2 ways to reach the top (take two 1-steps or take one 2-step). Otherwise, split your tree recursive call into two options:

  • Take a 1-step, and count the number of ways to traverse the n-1 remaining steps.
  • Take a 2-step, and count the number of ways to traverse the n-2 remaining steps.
