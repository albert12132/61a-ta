Conceptual

1) Consider what would happen if we took out the new function in the make_class function. Would we still be able to create instances of classes? If so, how would we do it?

2) What would happen if we changed the get_value function in make_instance to the following (changes are bolded): 

    def get_value(name):
        if name in attributes:
            return attributes[name]
        else:
            value = cls['get'](name)
            return value

How would this change the way we call "methods"?

3) What would happen if the attributes dictionary passed to the make_class function did not contain an __init__ method? Consider two cases: 1) base_Class is None; 2) base_class is something other than None.

4) What would happen if we changed the get_value function in make_instance to the following (changes are bolded):

    def get_value(name):
        if name in attributes:
            return attributes[name]
        else:
            return None

Code writing

1) Translate the following code, which uses Python's built-in class system, into code that uses our below-the-line OOP implementation.

    class Shop:
        def __init__(self, inventory):
            self.inventory = inventory

        def sell(self, item):
            if item in self.inventory and self.inventory[item] > 0:
                self.inventory[item] -= 1
                return item
            else:
                print("No " + item + " in inventory!")
                return None

    class AppleStore(Shop):
        def sell(self, item):
            product = Shop.sell(self, item)
            if product == None:
                print("Would you like to preorder?")

        def fix_product(self, item):
            if item in self.inventory:
                print("We can fix that!")
            else:
                print("Sorry!")
